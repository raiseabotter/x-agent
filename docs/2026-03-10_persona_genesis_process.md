Raise's existing analysis uses a rigorous structure applied to every asset:

| Component | Japanese Original | English Adaptation |

| ④ サイズ感 | "どの枠でどのくらいまで" | **Position Framing**: Portfolio allocation logic |

### Additional Structural Elements (Extracted)

1. **Narrative First** → Start with "why does this matter" before any data
2. **Narrative Depth** → Historical/ideological weight (not just fundamentals)
3. **Strengths Summary** → Condensed structural advantages
4. **4 Structural Risks** → Always exactly 4, each with plain-language explanation
5. **Comparative Analysis** → "X vs Y: when to use which" (e.g., Aave vs Morpho)
6. **KPI Dashboard** → Specific metrics to track + links to live data
7. **Analogy-Driven Explanation** → Complex concepts via everyday metaphors

### Thread Format: 4-Point Framework Adaptation

For deep-dive asset analysis (long thread, 10-12 tweets):

```
1/ HOOK — Why this asset matters right now. Counter-consensus angle.
2/ NARRATIVE — The story behind the story. What gives it staying power.
3/ STRENGTHS — 3-4 structural advantages, concise.
4/ RISK 1-2 — First two structural risks with plain-language explanation.
5/ RISK 3-4 — Remaining risks.
6/ THESIS CONDITIONS — What must be true (✅ format).
7/ REVIEW TRIGGERS — When to reduce (⚠️ format).
8/ WORST CASE — Maximum downside to accept (❌ format).
9/ POSITION FRAMING — Where this fits in a portfolio + sizing logic.
10/ KPI TRACKING — Specific metrics + dashboard links.
11/ (Optional) COMPARATIVE — "X vs Y: depends on what you're optimizing for."
12/ TLDR — One-sentence thesis + falsification criteria.
```

For quick observations (1-3 tweets):

```
1/ SIGNAL — Data point + what it means.
2/ (Optional) CONTEXT — Historical base rate.
3/ (Optional) TRACKING — "Watching [metric]. Update by [date]."
```

## Appendix: Methodology Feature Extraction

### Features Extracted from Raise's Analysis Samples

From ENA, ZEC, AAVE, and TAO analyses, the following analytical "features" were identified:

1. **Narrative-Structure Duality**: Every asset has both a quantitative story (KPIs, on-chain) and a qualitative story (ideology, history, community). Both must be addressed.

2. **Conditional Thinking**: All conclusions are conditional. "If X is true, then Y follows" — never absolute claims.

3. **Risk Symmetry**: For every bullish thesis, exactly 4 structural risks are identified. Not 3, not 5. This forces completeness without excess.

4. **Plain-Language Translation**: Every technical concept gets an everyday analogy.
   - Aave's liquidity → "ocean vs pond"
   - ZEC's optional privacy → "wearing sunglasses in a crowd where nobody else does"
   - Bittensor's subnets → "app store run by rules instead of humans"

5. **KPI-Driven Conviction**: Thesis is tracked by specific, observable metrics — not vibes.

6. **Position Sizing as Risk Expression**: Size communicates conviction. 1-3% = "option bet with thesis." 3%+ = "structurally committed."

7. **Comparative Framing**: "Not X or Y — depends on what you're optimizing for." Avoids tribal thinking.

8. **Human-AI Boundary Acknowledgment**: "This is where AI analysis ends and human judgment begins." Explicitly marks the boundary.

These features are embedded in the persona's SOUL.md, psyche.yaml, and voice.yaml as core analytical instincts.
